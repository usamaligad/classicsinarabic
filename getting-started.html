Overview

These instructions walk you through how to use the Alpheios Simple Site template and GitHub to create your own Alpheios-Enabled website.

Minimum Prerequisities

  1. An account on GitHub
  2. Very basic HTML skills

Technologies Used

Following the instructions below will enable you to create a very simple website. However, much more is possible than is described here if you delve into the technologies used to create the site. You can also create menus to navigate your site, add images and multimedia, use custom styles, link syntax trees in Alpheios and more.

  • The repository is hosted by GitHub. Knowing how to use GitHub is helpful for troublshooting problems with adding and editing files.
  • The website is served by GitHub Pages. Knowing more about this can help if you run into problems with the way the site functions or if it doesn't update as you expected.
  • The website is built using Jekyll. This is important if you want to add navigation and other typical website features.
  • The website's css theme is Cayman. This is important if you want to change the look and feel of the site.
  • The Alpheios functionality is provided by the Alpheios Embedded Library. You can explore additional Alpheios customization and feature options.

Instructions

  1. Create your site
    1. Login in to GitHub
    2. Go to this repository https://github.com/alpheios-project/simple-site
    3. Click the Green Use This Template button.
    4. Enter a name for your site's repository.
    5. Add a description of your planned site contents.
    6. Choose Public (if you choose Private you will not be able to publish it as a website)
    7. Click the green Create repository from Template button.
    8. You will be on your new repository page in GitHub.
    9. Click the Settings tab.
    10. Scroll down to the GitHub Pages section of the page.
    11. Select master branch from the Source dropdown.
    12. Click the Code tab.
    13. Click on the file named _config.yml.
    14. Click the Edit icon (it is a picture of pencil).
    15. Replace the text Alpheios Simple Site with the title of your site.
    16. Replace the text A Template for A Simple Website with Alpheios Embedded with a description of your site.
    17. Scroll to the bottom of the page to Commit Changes and enter a message to describe your change to this file. (e.g. something like "Updated site metadata".)
    18. Make sure the option Commit directly to the master branch is selected
    19. Click the Green Commit changes button.
    20. At this point, your GitHub web page should be published and updated at https://(youraccountname).github.io/(yoursitename). For example, if your account name is "janedoe" and your site name is "my-latin-site" the site will be published at https://janedoe.github.io/my-latin-site . As we have not yet made any changes to the content, it will contain the exact same contents as the template. (Note: it may take a minute or two for the page to update.)
  2. Edit your Site's home Page
    1. Click the Code tab.
    2. Click on the file named index.html. This file contains the main page of the site. You will replace the default contents with your own.
    3. Click on the Edit icon
    4. Replace the text "Simple Alpheios Enabled Site Template" with whatever you want the title of your main page to be.
    5. Delete everything from line 5 (the line after second ---) to the end.
    6. Add some text to your page. (See below for instructions on how to enable specific sections of text for Alpheios.)
    7. Scroll to the bottom of the page to Commit Changes and enter a message to describe your change to this file. (e.g. something like "Updated home page")
    8. Make sure the option Commit directly to the master branch is selected
    9. Click the Green Commit changes button.
    10. Your GitHub web page should be published and updated. (Note: it may take a minute or two for the page to update.)
  3. Enable Text For Alpheios

    1. Enable individual text sections on a page for Alpheios by specifying a class attribute on the surrounding HTML tag: class="alpheios-enabled" as well as a lang attribute set to the 3 character language code for your text (lang="lat" for Latin, lang="grc" for Greek, lang="ara"> for Arabic and lang="per" for Persian.)

    2. Enable an entire page text for a single language by specifying use of the corresponding language template in the metadata at the top of the file.

      Use the files latin.html, greek.html, arabic.html or persian.html in this directory or see the Example Latin Page File Contents below:

  4. Add new files to your site by using the "Add File" button when on the code tab. You can create a new file or upload a file from your local file system.

    Make sure to use only HTML fragments for your files. The page contents following the metadata section should be marked up as HTML fragments. They will be inserted automatically in the body element of the page. Do not use full HTML files.

  5. Update the descriptive metadata for your page.

    Metadata fields are set at the top of the page in the metadata section which is preceded and followed by lines containing only ---.

    At a minimum you must specify layout and title for each page. layout should be set to default if your page has text in multiple languages (such as instructional text in English and reading text in Latin). Otherwise set it to the language of your text (latin, greek, arabic or persian) as described above.title should be set to whatever you want your page title to be.

    NOTE: It is very important that you have a single space after the colon that separates the metadata field name from the field contents. Otherwise the site will fail to build correction.

Troubleshooting

If your changes to a page fail to appear in the published site, check to see if the commit succeeded by doing the following:

  1. Click on the Code tab and look to see if the last commit listed at the top of the directory listing has a green ✓ next to it. If it does, the try clearing your browser cache and reloading the page. However, if instead it has a red ❌ it means that the page build failed. A common cause is invalid metadata field specification (e.g. a missing space).
  2. You can also check the history of your GitHub Pages build. Click on the Code tab, and on the right side of the page you will see a section entitled Environments. Click the github-pages link and you'll see the history of the page deployments.
